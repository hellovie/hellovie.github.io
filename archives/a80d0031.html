动态规划解题技巧总结

动态规划是把原问题分解为相对简单的子问题的方式求解复杂问题的方法。在用动态规划解题时，应该注意先解题再优化，要把解题放在第一位，这里的解题不做优化。

解题步骤一：确定状态

确定状态需要三个步骤：

  1. 最后一步：研究最优策略的最后一步，一般是列举最后一步的所有可能进行分析。
  2. 子问题：通过最后一步与倒数第二步或第三步直接的关系发现规律，化为子问题。
  3. 状态数组：根据子问题来创建状态数组。（一般子问题有几个变量就是几维数组）

我们最终目的是要确定状态数组的每个元素表示什么。（状态数组通常是 dp[i][j]）

解题步骤二：设计状态转移方程

状态转移方程，是动态规划中本阶段的状态往往是上一阶段状态和上一阶段决策的结果。

换言之就是设计出一个递推公式。

解题步骤三：初始条件和边界情况

初始条件：用状态转移方程无法计算出来，但我们又需要它的定义。即给出递推公式的基本条件。

边界情况：根据题目和状态转移方程判断边界情况，剔除出界情况。

解题步骤四：计算顺序

状态转移方程是动态规划中本阶段的状态往往是上一阶段状态和上一阶段决策的结果，所以计算顺序要利用之前得出的结果。一般是从小到大，从上到下，从左到右。


例题 1：蓝桥杯印章

题目要求

时间限制：1.0s

内存限制：256.0MB

问题描述：共有 n 种图案的印章，每种图案的出现概率相同。小A买了 m 张印章，求小A集齐 n 种印章的概率。

输入格式：一行两个正整数 n 和 m。

输出格式：一个实数P表示答案，保留4位小数。

样例输入：2 3

样例输出：0.7500

数据规模和约定：1≤n，m≤20

题目链接：http://lx.lanqiao.cn/problem.page?gpid=T3002

题目分析

我们现在有 n 种图案，买了 m 张，集齐概率为 Pm，我们买的每张印章概率都为 1/n，我们先假设 m >= n ，如果 m < n 的话是不可能集齐 n 种的。如果要暴力解题，就需要遍历所有可能再求概率，但是我们发现已经买 n - 1张集齐了 k 种的概率可以被买 n 张集齐了 k 种的情况或者买 n 张集齐了 k + 1 种的情况使用，那么我们就可以用一个数组来记录之前求过的概率，减少重复运算，就可以用动态规划解题。

  • 确定状态

  1. 最后一步：买第 m 张的印章，这时第 m 张可能是不重复的印章也可能是重复的印章。

  2. 子问题：

    • 如果第 m 张是不重复的话，Pm = Pm - 1 * (1/n)；因为第 m - 1 张已经集齐了 n - 1 种，所以第 m 张要抽到唯一 1 张没抽过的；
    • 如果第 m 张是重复的话，Pm = Pm - 1 * 1，因为第 m - 1 张已经集齐了 n 种，所以第 m 张不管是抽到哪种都不影响；
    • 那么现在问题就变成求 Pm -  1，即买 m - 1 张集齐 n 种或买 m - 1 张集齐 n - 1种；
    • 我们发现再以此类推，存在两个变量，那么得出子问题就是求买 i 张，集齐 j 张的概率。

  3. 状态数组：用 dp[i][j] 表示买了 i 张，集齐 j 种的概率。

  • 设计状态转移方程

(n - j + 1) / n：表示在 n 种印章中，抽出剩下的 n - j + 1 种印章的概率。

(j / n)：表示在 n 种印章中，抽出已抽的 j 种印章的概率。

  1. 买第 i 张时集齐 j 张的概率（没有重复）：dp[i][j] = dp[i - 1][j - 1] * (n - j + 1) / n。

  2. 买第 i 张时集齐 j 张的概率（有重复）：dp[i][j] = dp[i - 1][j] * (j / n)。

  3. dp[i][j] = dp[i - 1][j - 1] * (n - j + 1) / n + dp[i - 1][j] * (j / n)。

  • 初始条件和边界情况
  1. 当 i < j 时，无法买 i 张去集齐 j 张，这时 dp[i][j] = 0；
  2. 当 j = 1，i >= 1，表示买 i 张去集齐 1 张；已知每张印章概率都为 1/n，共有 n 种印章，所以在 n 种印章中抽 i 张，抽中的都是同一种印章的概率为：dp[i][j] = (1/n)i * n。
  • 计算顺序

从小到大，因为后面的概率需要用到前面得出的结果。

例题 2：数字三角形

题目要求

时间限制：1.0s

内存限制：256.0MB

问题描述：（图３.１－１）示出了一个数字三角形。 请编一个程序计算从顶至底的某处的一条路径，使该路径所经过的数字的总和最大。

  • 每一步可沿左斜线向下或右斜线向下走；

  • 1＜三角形行数≤100；

  • 三角形中的数字为整数0，1，…99；　

    　
    　（图３.１－１)

输入格式：文件中首先读到的是三角形的行数。接下来描述整个三角形

输出格式：最大总和（整数）

样例输入：

5
7
3 8
8 1 0
2 7 4 4
4 5 2 6 5

样例输出：30

题目链接：http://lx.lanqiao.cn/problem.page?gpid=T312

题目分析

计算从第 1 层到第 n 层的一条路径，求数字总和最大值。如果用暴力解法的话，我们就需要遍历每一条路径，再求最大值，时间复杂度高。我们发现如果从第 1 层走到第 n 层的第 k 个数或第 k + 1 个数的数字总和都需要用到从第 1 层走到第 n - 1 层的第 k 个数的数字总和，那么我们就可以用一个数组来保存之前的数字总和，减少重复运算，就可以用动态规划解题。

  • 确定状态
  1. 最后一步：假设用一个 二维数组 M[][] 来表示三角形每个数字的位置，那么最顶层的那个数就是 M[1][1]。假设倒数第二步就是 M[n - 1][j]，那么最后一步就可能是M[n][j] 或 M[n][j + 1]（每一步可沿左斜线向下或右斜线向下走）。
  2. 子问题：
    • 那么，从第 n - 1 层的 j 处到第 n 层的数字总和最大值 = M[n - 1][j] + Max{M[n][j], M[n][j + 1]}，以此类推，可以得出第 n - 1 层每个数到第 n 层的数字最大总和；
    • 那么问题就变成了从第 1 层到第 n - 1 层的一条路径，求数字总和最大值；
    • 我们又可以得出第 n - 2 层每个数到第 n - 1 层的数字最大总和，第 n - 2 层的对应数到第 n 层的数字最大总和 = 第 n - 2 层对应数 + 第 n - 1 层对应数到第 n 层的数字最大总和（上面得出的），从而得出从第 n - 2 层到第 n 层的数字总和最大值；
    • 不断递推，我们每次都能得出第 i 层的第 j 个数到第 n 层的数字总和最大值。
    • 那么子问题就是求第 i 层的第 j 个数到第 n 层的数字总和最大值。
  3. 状态数组：那么我们就用 dp[i][j] 表示第 i 层的第 j 个数到第 n 层的数字总和最大值。
  • 设计状态转移方程

M[i][j]：三角形每个数字的位置。

dp[i][j]：第 i 层的第 j 个数到第 n 层的数字总和最大值。

dp[i][j] = Max(dp[i + 1][j], dp[i +1][j]) + M[i][j]。

  • 初始条件和边界情况
  1. dp[n][j] = M[n][j]；
  2. 因为第 n 层 dp 已经初始化，所以 n > i >= 1；
  3. i >= j >= 1。
  • 计算顺序

自底向上。

总结

  • 首先分析问题中是否存在许多重复计算，能够使用动态规划。

  • 其次按照动态规划解题技巧总结：

    • 确定状态：

      • 最后一步；

      • 子问题；

      • 状态数组。

    • 设计状态转移方程；

    • 初始条件和边界情况；

    • 计算顺序。

  • 最后优化空间。

  • 记住动态规划类型题只能通过多做来熟悉。
